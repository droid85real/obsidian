### Key Uses

1. **Caching**
    - Store frequently accessed data
    - Reduces load on DB
2. **Rate Limiting**
    - Control API usage
    - Prevent spam / abuse
3. **Distributed Locking**
    - Prevent race conditions
    - Used in payments, booking systems
4. **Message Queue / Jobs**
    - Background processing via  
        BullMQ
    - Example: video processing
5. **Real-time (Pub/Sub)**
    - Chat, notifications, live updates
6. **Session Store**
    - Store user sessions across servers
    - Needed for scaling
7. **Leaderboards (Sorted Sets)**
    - Rankings, scores
8. **Temporary State**
    - Upload progress, processing status
    - Short-lived system data
9. **Feature Flags / Config**
    - Toggle features without redeploy
